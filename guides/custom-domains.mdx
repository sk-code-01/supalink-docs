## Overview

You can point your own domain (or subdomain) to any SimpleHost site. SSL certificates are provisioned automatically.

| Plan | Custom Domains |
|------|---------------|
| Free | 1 |
| Hobby ($5/mo) | 5 |

## Setup

### Step 1: Add the domain in your dashboard

1. Go to [simplehost.dev/dashboard](https://simplehost.dev/dashboard)
2. Click the **Domains** tab
3. Click **+ Add Domain**
4. Enter your domain (e.g., `docs.mysite.com`)

### Step 2: Configure DNS

Add these DNS records at your domain registrar or DNS provider:

#### CNAME Record

| Type | Name | Target |
|------|------|--------|
| CNAME | `docs` (or your subdomain) | `proxy.simplehost.dev` |

#### TXT Record (verification)

| Type | Name | Value |
|------|------|-------|
| TXT | `_simplehost.docs` (or `_simplehost.yoursubdomain`) | The verification code shown in the dashboard |

### Step 3: Wait for verification

DNS changes can take up to 48 hours to propagate, but typically complete within a few minutes. The domain status in your dashboard will change from **Pending** to **Active** once verified.

## Using a Root Domain

For root/apex domains (e.g., `mysite.com` without a subdomain), you need a DNS provider that supports CNAME flattening or ALIAS records:

- **Cloudflare** — Supports CNAME flattening (use CNAME at root)
- **Route 53** — Use an ALIAS record
- **DNSimple** — Use an ALIAS record

Not all DNS providers support this. If yours doesn't, use a subdomain like `www.mysite.com` instead.

## SSL Certificates

SSL certificates are provisioned and renewed automatically through Cloudflare. You don't need to do anything — HTTPS just works once the domain is verified.

## Managing Domains from the Terminal

You can manage everything from the terminal -- no dashboard required.

### Add a domain

```bash
curl -X POST https://simplehost.dev/api/v1/domains \
  -H "Authorization: Bearer sh_live_your_key_here" \
  -H "Content-Type: application/json" \
  -d '{"domain": "docs.mysite.com"}'
```

The response includes the exact DNS records you need to configure.

### Verify DNS records

After configuring DNS at your registrar, verify:

```bash
curl -X POST https://simplehost.dev/api/v1/domains/docs.mysite.com/verify \
  -H "Authorization: Bearer sh_live_your_key_here"
```

### List your domains

```bash
curl -X GET https://simplehost.dev/api/v1/domains \
  -H "Authorization: Bearer sh_live_your_key_here"
```

### Remove a domain

```bash
curl -X DELETE https://simplehost.dev/api/v1/domains/docs.mysite.com \
  -H "Authorization: Bearer sh_live_your_key_here"
```

See the [Domains API reference](/api-reference/domains) for full request/response details.
